# Инструкция: получение ID приложения и защищённого ключа ВКонтакте

Для работы входа через ВКонтакте и серверной проверки участия в закрытой группе нужны **3 значения**:

> Важно: саму проверку участия в группе сайт делает через `access_token` и `VK_GROUP_ID`.  
> `VK_APP_ID` и `VK_APP_SECRET` нужны для безопасного получения этого `access_token` при входе через ВК.

| Параметр | Что это | Где взять |
|----------|---------|-----------|
| `VK_APP_ID` | ID приложения | Настройки приложения VK |
| `VK_APP_SECRET` | Защищённый ключ (секретный ключ) | Настройки приложения VK |
| `VK_GROUP_ID` | ID группы ВК | Адресная строка группы |

---

## Шаг 1. Создание приложения VK ID

1. Перейдите на страницу подключения авторизации VK ID:
   - **https://dev.vk.com/ru/admin/connect-auth**
2. Если страница не открылась напрямую, зайдите на **https://dev.vk.com/** и откройте раздел создания приложения для авторизации.
3. Нажмите **«Создать приложение»**.
4. Укажите:
   - **Название** — например: `Портфолио Студентов`
   - **Платформа** — выберите **`Web`**
5. Подтвердите создание приложения.

> После создания вы попадёте на страницу настроек приложения.

---

## Шаг 2. Настройка авторизации

### 2.1. Основные настройки

На странице приложения найдите раздел **«Подключение авторизации»** или блок с настройками авторизации.

1. **Базовый домен** — укажите: `apparchi.ru`
2. **Доверенный Redirect URL** — добавьте:
   ```
   https://apparchi.ru/auth/vk/callback
   ```
3. Нажмите **«Сохранить»** или **«Готово»**

### 2.2. Запишите ключи

После сохранения найдите и скопируйте:

- **ID приложения** — числовой ID (например: `52145678`) → это `VK_APP_ID`
- **Защищённый ключ** (или «Секретный ключ») → это `VK_APP_SECRET`

> Защищённый ключ может быть скрыт за звёздочками — нажмите «Показать», чтобы скопировать.

---

## Шаг 3. Получение ID группы ВКонтакте

ID группы нужен для проверки: авторизоваться смогут только участники вашей группы.

### Способ A — если у группы числовой адрес:

Откройте группу в браузере. Если адрес выглядит так:
```
https://vk.com/club123456789
```
то ID группы = **123456789** (цифры после `club`)

### Способ B — если у группы текстовый адрес (короткое имя):

Например, адрес: `https://vk.com/my_beauty_club`

В таком случае просто отправьте разработчику **ссылку на группу**, например:

```
https://vk.com/my_beauty_club
```

Числовой ID группы мы определим сами.

---

## Шаг 4. Передайте данные разработчику

Отправьте разработчику эти 3 значения:

```
VK_APP_ID=_______
VK_APP_SECRET=_______
VK_GROUP_ID=_______
```

---

## Как это работает (для понимания)

```
Пользователь нажимает «Войти через ВК»
       ↓
ВКонтакте открывает страницу входа и подтверждения доступа
       ↓
После успешного входа ВК возвращает пользователя на сайт
       ↓
Сервер сайта получает временный код авторизации
       ↓
Сервер обменивает его на access_token с помощью:
APP_ID + защищённого ключа + redirect URI
       ↓
Сайт проверяет через VK API: состоит ли пользователь в закрытой группе?
       ↓
  ДА → Вход в личный кабинет
  НЕТ → «Вы не являетесь участником клуба»
```

Авторизоваться смогут **только участники** указанной группы ВК.

---

## Важные моменты

- **Redirect URL должен совпадать точно** — `https://apparchi.ru/auth/vk/callback` (с `https`, без слэша в конце)
- **Защищённый ключ нельзя публиковать** — передавайте его только разработчику лично (не в открытых чатах)
- Если вы смените домен сайта, нужно будет обновить Redirect URL в настройках приложения ВК

## Частые вопросы

**Q: Можно ли использовать существующее приложение ВК?**
A: Да, если у вас уже есть приложение для авторизации с платформой `Web` и вы можете добавить в него нужный Redirect URL.

**Q: Что если студент не состоит в группе?**
A: Он увидит сообщение «Вы не являетесь участником клуба. Вступите в группу и попробуйте снова.»

**Q: Приложение ВК платное?**
A: Нет, создание и использование приложений ВКонтакте бесплатное.

**Q: Что именно нужно разработчику?**
A: Обычно достаточно `VK_APP_ID`, `VK_APP_SECRET` и `VK_GROUP_ID`. Дальше сайт сам получит `access_token` через VK OAuth и проверит участие пользователя в группе.
