# Инструкция: вход по одноразовой ссылке после проверки группы ВК

## Как это работает для студента

1. Студент пишет в сообщения сообщества ВК, например: `кабинет`
2. n8n получает `vk_id` отправителя из входящего сообщения
3. n8n проверяет через VK API, состоит ли этот `vk_id` в закрытой группе
4. Если студент не состоит в группе:
   - ссылка не выдается
   - студенту отправляется сообщение: `Доступ к кабинету есть только у участников закрытой группы`
5. Если студент состоит в группе:
   - n8n вызывает внутренний endpoint проекта
   - проект создает одноразовую ссылку
   - n8n отправляет студенту эту ссылку в личные сообщения
6. Студент открывает ссылку
7. Ссылка срабатывает только один раз:
   - при первом открытии создается обычная web-сессия
   - при повторном открытии та же ссылка уже недействительна
8. После входа студент попадает в личный кабинет и может загружать фото

---

## Почему этот вариант подходит

- Не требует проблемного VK OAuth через Госуслуги или звонок
- Доступ получает только владелец конкретного `vk_id`
- Ссылка не живет постоянно и не может использоваться бесконечно
- Перед выдачей ссылки membership в закрытой группе проверяется заново

---

## Что хранится в базе

### `users`

- `vk_id` — стабильный идентификатор пользователя ВК
- `is_group_member` — результат последней проверки membership
- `last_vk_check_at` — время последней проверки
- стандартные поля пользователя, тарифа и статуса

### `login_tokens`

- одноразовые токены входа
- у токена есть срок жизни
- после первого использования токен помечается как использованный
- при выпуске новой ссылки предыдущая активная ссылка для этого пользователя отзывается

---

## Внутренний endpoint для n8n

`POST /auth/internal/issue-link`

### Заголовки

`X-Internal-Token: <INTERNAL_API_TOKEN>`

### JSON body

```json
{
  "vk_id": 123456789,
  "name": "Иван Иванов",
  "tariff": "УВЕРЕННЫЙ",
  "photo_url": "https://...",
  "is_group_member": true
}
```

### Ответ при успехе

```json
{
  "ok": true,
  "issued": true,
  "login_url": "https://apparchi.ru/auth/link?token=...",
  "expires_at": "2026-03-19T18:35:00+00:00",
  "vk_id": 123456789,
  "user_id": 7
}
```

### Ответ, если пользователь не в группе

```json
{
  "ok": false,
  "issued": false,
  "reason": "not_group_member",
  "message": "Пользователь не состоит в закрытой группе ВК."
}
```

## Рекомендуемый сценарий в n8n

1. `VK Trigger` или входящий webhook из ВК
2. Извлечь `from_id`
3. Выполнить проверку membership через VK API
4. Если membership = `false`:
   - отправить отказное сообщение
   - завершить сценарий
5. Если membership = `true`:
   - вызвать `POST /auth/internal/issue-link`
   - получить `login_url`
   - отправить студенту сообщение со ссылкой

---

## Что важно настроить

В `.env` проекта должны быть:

```env
SESSION_SECRET=...
INTERNAL_API_TOKEN=...
ONE_TIME_LINK_TTL_MINUTES=30
```

---

## Админский сценарий

В админке можно вручную выпустить одноразовую ссылку только для:

- активного пользователя
- подтвержденного участника группы ВК
- или администратора

Это нужно как резервный способ поддержки, если автоматическая отправка через n8n временно недоступна.

---

## Ограничения

- Если студент переслал уже использованную ссылку, она не откроется повторно
- Если срок жизни ссылки вышел, нужно запросить новую
- Если студент вышел из группы ВК, новая ссылка ему уже не будет выдана
- Если сессия уже создана, она действует до окончания `SESSION_TTL_HOURS`

---

## Что запускать при обновлении старой базы

Если база уже существует, нужно один раз выполнить:

```bash
python scripts/migrate_one_time_links.py
```

Это добавит нужные поля в `users` и создаст таблицу `login_tokens`.
